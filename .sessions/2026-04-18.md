## What Was Built

- `js/physics/vector.js` — `Vector2D` class (immutable). Methods: `add`, `sub`, `scale`, `dot`, `magnitude`, `normalize` (zero-safe), `angle` (degrees), `closestPointOnSegment`, `distanceToSegment`, `static fromAngle`. `closestPointOnSegment` added now (one line over plain `distanceToSegment`) so Phase 4 anti-tunneling can interpolate the exact capture point on the segment.
- `js/physics/constants.js` — re-exports `K_COULOMB, R_MIN_M, R_CAPTURE_M, UC_TO_C, NC_TO_C, UG_TO_KG, PX_TO_M, PIXELS_PER_CM` from `config.js`. Decouples physics from UI/canvas constants.
- `js/physics/engine.js` — `FieldEngine.computeFieldAt(x_px, y_px, charges)` → `{ex, ey, magnitude, angle_deg, pos_cm}`. Signed-Q superposition (no manual r̂ flip). `r²` clamped to `R_MIN_M²`; r̂ uses raw r so it stays a true unit vector. `formatField(magnitude)` produces scientific notation with Unicode superscripts (`'8.99 × 10⁵'`, `'8.99 × 10⁻³'`).
- `js/physics/energy.js` — `computePotentialEnergy(particle, charges)` (pair-sum Coulomb, joules) and `computeKineticEnergy(particle)` (½mv², joules).
- `js/physics/grid-worker.js` — Web Worker, self-contained (inlined constants + Coulomb math, no ES6 imports). Centered cells: `x = spacingPx/2 + i·spacingPx`. At spacing=25 → 48×32 = 1,536 points. Returns `{grid, maxMag, minMag}`.
- `js/main.js` — Phase 1 validation bootstrap (will be replaced in Phase 3). Runs 5 checks on page load.
- `.gitignore` — added `.playwright-mcp/` (browser-automation test artifacts).

## Verified

Browser: `python -m http.server 8080` → `http://localhost:8080` in Chromium via Playwright MCP. Console output:

```
[Phase 1] worker: grid.length=1536, maxMag=2.248e+7, minMag=1.815e+4
[Phase 1] ✅ validations passed
```

All 5 validations pass:
1. Single charge: Q=+1 μC at (600,400)px, probe (700,400)px → `|E| = 8.99×10⁵ V/m`, `Eₓ = 8.99×10⁵`, `E_y ≈ 0`, `θ ≈ 0°`.
2. Dipole symmetry: ±5 μC at (450/750, 400)px, probe midpoint → `|E| ≈ 4.00×10⁶ V/m`, `Eₓ > 0`, `θ ≈ 0°` (proves no sign-flip bug on negative charge).
3. `formatField(899000)` → `"8.99 × 10⁵"`; `formatField(0.00899)` → `"8.99 × 10⁻³"` (negative exponent path).
4. Energy: q=+1 nC at 10 cm from +1 μC → `U = 8.99×10⁻⁵ J`, `K = 0`.
5. Worker round-trip: 1,536-point grid returned within one frame, `maxMag/minMag` valid.

No console errors, no NaN, no module-resolution failures.

## Open Items

- `.playwright-mcp/` artifacts directory exists from this session's verification — now gitignored.
- Engine uses `r̂ = r_vec / r_raw` (true unit vector) rather than spec literal `r_vec / r_clamped`. This is the standard physics convention; affects no tests. Documented here in case Phase 2 renderer expects different.
- Worker constants (`K_COULOMB`, `R_MIN_M`, `UC_TO_C`, `PX_TO_M`) are duplicated from `config.js` because Workers can't use ES6 imports. **Divergence risk**: any change to these in `config.js` must be mirrored in `grid-worker.js`. Add a code comment in Phase 2 if a build-time inline becomes warranted.
